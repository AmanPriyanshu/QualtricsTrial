Project Documentation Overview

Explore the different implementations of our privacy consent mechanisms. Each version aims to test various aspects of user interaction and understanding.

Description File Explanation Reasoning
Dynamic Hierarchical One Page (Descriptive with Points) View File Dynamic rendering allows for interactive, on-the-fly adjustments to content displayed based on user selection. This hierarchical setup organizes information into grouped categories with headers, with detailed point-wise descriptions for each category and function. This format tests user understanding and preference for interactive, detailed, and well-structured consent mechanisms. Point-wise details may improve comprehension and retention of information regarding their data sharing consent.
Dynamic Hierarchical One Page (Descriptive) View File Similar to the previous, this format uses dynamic rendering and hierarchical information structuring on a single page. However, it presents information in a descriptive paragraph format rather than points. By contrasting with point-wise description, this version assesses how narrative-style information affects user engagement and understanding, highlighting preferences between different descriptive formats in dynamic settings.
Non-Dynamic Hierarchical Split Page (Descriptive with Points) View File This version features a split-page layout that statically displays content. Users navigate between two separate pages for category and function selections, with hierarchical grouping and point-wise descriptions. This setup evaluates user response to traditional multi-page forms that require navigation. It tests if splitting content across pages affects comprehension or preference, compared to a single dynamic page.
Non-Dynamic Hierarchical Split Page (Descriptive) View File Employing a non-dynamic, two-page approach, this layout also uses hierarchical structuring but with narrative-style descriptions. It separates category and function choices across pages. This format allows testing of user preferences for non-dynamic content delivery in a descriptive format, evaluating whether users favor narrative information over point-wise in a static, multi-page context.
(Default) Non-Dynamic Raw List Split Page (Descriptive) View File This version displays a non-dynamic raw list on split pages without hierarchical grouping. Information is presented in a straightforward descriptive format across multiple pages. The aim is to test user navigation and understanding in a basic list format, determining if the absence of hierarchical grouping affects the clarity and effectiveness of the consent process.
Non-Dynamic Raw List Split Page (Point Wise) View File Features a simple, non-dynamic list format split across pages, providing information in concise points. It lacks hierarchical structuring, focusing purely on content clarity. This setup tests whether a straightforward, bulleted list enhances user comprehension and consent accuracy compared to more structured formats, especially in a static, multi-page setup.
